## 1. 项目概览

### 项目架构

HelloVirtulCell (OpenVCell-MVP) 是一个虚拟细胞研究项目的最小可行性演示，实现了4阶段pipeline：

```
Stage 1: 数据底座 (合成/真实单细胞数据)
    ↓
Stage 2: 基础模型 (Transformer细胞嵌入)
    ↓
Stage 3: 扰动预测 (CPA-lite架构) ← 本文重点
    ↓
Stage 4: 机理耦合 (FBA代谢建模)
```

### 项目特点

- **数据规模**: 2000 cells × 200 genes
- **细胞类型**: 3种
- **扰动条件**: 5种 (control, KO_geneA, KO_geneB, drug_X, cytokine_Y)
- **模型架构**: Transformer + CPA-lite + Toy FBA
- **评估性能**: 
  - 基因级 Pearson 相关性: **0.592**
  - Top-20 DEG 召回率: **0.150**
  - 细胞类型 macro-F1: **0.998**

## 2. 扰动预测模块架构

### 核心思路

扰动预测要解决的问题：**给定一个细胞和一种扰动，预测这个细胞的基因表达会如何变化**。

```
输入: (64维细胞嵌入, 扰动类型ID) → 输出: 200维基因表达变化
```

### 数据流水线

#### 数据构建 (`build_pert_dataset`)
```python
def build_pert_dataset(adata, cell_embs):
    perts = ["control", "KO_geneA", "KO_geneB", "drug_X", "cytokine_Y"]
    pert2id = {p: i for i, p in enumerate(perts)}  # 扰动 → ID 映射
    
    # 返回: (细胞嵌入, 扰动ID, 表达向量, 扰动名称列表)
    return cell_embs, pert_ids, expressions, perts
```

#### 数据集类 (`PertDataset`)
```python
class PertDataset(Dataset):
    def __getitem__(self, i):
        return cell_emb[i], pert_id[i], expression[i]
```

每个样本是一个三元组：`(64维细胞嵌入, 扰动ID, 200维表达向量)`

### CPA-lite 模型架构

采用 **加法分解** 的思路：

```
预测表达 = 基础表达 + 扰动效应
```

#### 模型结构
```python
class CPAlite(nn.Module):
    def __init__(self, emb_dim=64, n_perts=5, n_genes=200, hidden=128):
        # 1. 基础解码器：细胞嵌入 → 基础表达
        self.basal = nn.Sequential(
            nn.Linear(emb_dim, hidden), 
            nn.GELU(), 
            nn.Linear(hidden, n_genes)
        )
        
        # 2. 扰动嵌入：每种扰动都有学习到的向量表示
        self.pert_emb = nn.Embedding(n_perts, emb_dim)
        
        # 3. 扰动解码器：(细胞嵌入 + 扰动嵌入) → 扰动效应
        self.pert_dec = nn.Sequential(
            nn.Linear(emb_dim * 2, hidden),
            nn.GELU(), 
            nn.Linear(hidden, n_genes)
        )
```

#### 前向传播逻辑
```python
def forward(self, cell_emb, pert_id):
    # 1. 基础表达：只看细胞类型，不考虑扰动
    basal = self.basal(cell_emb)
    
    # 2. 扰动向量：查表获得扰动的学习表示
    pert_vec = self.pert_emb(pert_id)
    
    # 3. 扰动效应：细胞+扰动的交互作用
    delta = self.pert_dec(torch.cat([cell_emb, pert_vec], dim=-1))
    
    # 4. 最终预测 = 基础 + 扰动
    return basal + delta, basal, delta
```

### Leave-One-Perturbation-Out (LOPO) 策略

测试模型对 **未见过扰动** 的泛化能力：

```python
def train_perturb(adata, cell_embs, heldout="drug_X"):
    # 将 "drug_X" 完全排除在训练集外
    train_mask = pert_ids != heldout_id
    test_mask = ~train_mask
    
    # 只用其他4种扰动训练
    train_ds = PertDataset(cells[train_mask], pert_ids[train_mask], expr[train_mask])
    test_ds = PertDataset(cells[test_mask], pert_ids[test_mask], expr[test_mask])
```

**关键特点**：模型从未见过 "drug_X" 的训练样本，但要预测 "drug_X" 对细胞的影响！

### 零样本预测机制

```python
def predict_expression(model, cell_emb, pert_id):
    with torch.no_grad():
        pred_expr, _, _ = model(cell_emb, pert_id)
    return pred_expr.numpy()
```

**为什么零样本预测可能有效？**

1. **细胞表示的泛化性**：Stage 2 学到的细胞嵌入捕获了细胞状态的通用特征
2. **扰动嵌入的学习**：每种扰动都有独立的向量表示，模型学会了扰动的"语义"
3. **加法组合性**：`basal + delta` 的设计假设扰动效应是可组合的

## 3. 扰动嵌入的底层逻辑

### 生物学动机：扰动的"身份卡"

#### 核心假设
```python
self.pert_emb = nn.Embedding(n_perts, emb_dim)
```

这行代码背后的生物学假设：**每种扰动都有自己独特的"分子指纹"或"作用模式"**。

**类比理解**：
- `KO_geneA` → 某个特定的基因调控网络失效
- `drug_X` → 某些特定的蛋白靶点被抑制  
- `cytokine_Y` → 某条信号通路被激活

每种扰动虽然作用机制不同，但都可以用一个 **64维向量** 来概括其"扰动特征"。

### 交互作用建模的数学原理

#### 为什么是 `[cell_emb, pert_vec]` 拼接？

```python
# 关键的交互建模
pert_vec = self.pert_emb(pert_id)              # 查表获得扰动向量
delta = self.pert_dec(torch.cat([cell_emb, pert_vec], dim=-1))  # 交互作用
```

这个设计体现了 **条件依赖** 的思想：

```
扰动效应 = f(细胞状态, 扰动类型)
```

**不是简单相加，而是条件交互**：
- 同样的药物 `drug_X` 作用在不同细胞类型上，效果可能完全不同
- 神经元细胞 vs 肌肉细胞对同一个基因敲除的响应会不一样

#### 具体的计算过程

```python
# 假设具体数值
cell_emb = [0.2, -0.1, 0.5, ...]     # 64维，表示某个T细胞的状态
pert_vec = [0.8, 0.3, -0.2, ...]     # 64维，表示"drug_X"的扰动特征  

# 拼接成 128 维
combined = [0.2, -0.1, 0.5, ..., 0.8, 0.3, -0.2, ...]

# 通过 MLP 学习交互
delta = MLP(combined)  # → 200 维基因扰动效应向量
```

### 网络架构的深层含义

#### 为什么是 `emb_dim * 2` → `hidden` → `n_genes`？

```python
self.pert_dec = nn.Sequential(
    nn.Linear(emb_dim * 2, hidden),  # 128 → 128，学习交互特征
    nn.GELU(),                       # 非线性激活
    nn.Linear(hidden, n_genes)       # 128 → 200，输出基因扰动
)
```

这个三层结构有特定的功能分工：

**第一层 (128→128)**：**特征融合层**
- 学习 `cell_emb` 和 `pert_vec` 之间的交互关系
- 相当于问：这个细胞状态 + 这种扰动 → 会产生什么样的"综合效应"？

**激活函数 (GELU)**：**非线性变换**  
- 生物系统是高度非线性的，简单的线性组合不够
- GELU 提供平滑的非线性，帮助学习复杂的生物学关系

**第二层 (128→200)**：**基因映射层**
- 将"综合效应"翻译成具体的基因表达变化
- 每个输出维度对应一个基因的扰动量 (Δexpression)

### 学习机制：如何习得扰动表示

#### Embedding 的自动学习

```python
# 训练前：随机初始化
pert_emb["drug_X"] = random_vector_64d

# 训练过程：通过梯度下降优化
for (cell, perturbation, true_expression) in training_data:
    predicted_delta = pert_dec(cell_emb, pert_emb[perturbation])
    loss = MSE(predicted_expression, true_expression)
    # 反向传播会同时优化 pert_emb 和 pert_dec
```

**关键洞察**：模型会自动学会每种扰动的"最优表示"，使得预测误差最小。

#### 扰动向量学到了什么？

假设训练后 embedding 学到了这样的模式：

```python
pert_emb["control"]    = [0.0, 0.0, 0.0, ...]   # 接近零向量
pert_emb["KO_geneA"]   = [0.8, -0.2, 0.1, ...]  # 某种调控模式
pert_emb["drug_X"]     = [-0.3, 0.9, 0.4, ...]  # 另一种作用模式  
pert_emb["cytokine_Y"] = [0.2, 0.1, -0.7, ...]  # 信号通路模式
```

每个维度可能对应某种潜在的生物学"轴"：
- 第1维：细胞周期相关？
- 第2维：凋亡相关？  
- 第3维：代谢相关？
- ...

### 零样本预测的奥秘

#### 为什么能预测未见过的扰动？

这是最神奇的地方！看起来矛盾：如果 `drug_X` 从未在训练中出现，`pert_emb["drug_X"]` 怎么能有意义？

**答案在于 embedding 空间的结构化学习**：

```python
# 训练时只见过这4种扰动
seen = ["control", "KO_geneA", "KO_geneB", "cytokine_Y"]

# 但网络学会了一般化的 cell-perturbation 交互函数
def interaction_function(cell_state, perturbation_vector):
    return some_learned_mapping(cell_state, perturbation_vector)
```

**关键假设**：虽然 `drug_X` 的 embedding 是随机初始化的，但：
1. **交互函数已经学会了** 如何处理 (cell, perturbation) 对
2. **随机的扰动向量仍然可能产生合理的交互**
3. **embedding 空间的连续性** 使得邻近向量产生相似效果

### 设计哲学：组合性与可解释性

#### 组合性原理
```python
final_expression = basal_expression + perturbation_delta
```

这个设计隐含地假设：
- **细胞有一个基础状态** (basal)
- **扰动产生可加性的调整** (delta)  
- **不同扰动效应可以线性叠加** (虽然当前只处理单一扰动)

#### 可解释性优势

```python
pred, basal, delta = model(cell_emb, pert_id)
```

模型输出三个部分，便于分析：
- `basal`：这个细胞的"自然"表达模式
- `delta`：扰动引起的"变化量"  
- `pred`：最终的预测结果

科学家可以单独研究扰动效应，而不被基础表达混淆。

## 4. MLP交互学习机制

### 数学层面：权重矩阵的作用

#### 第一层线性变换 (128 → 128)

```python
nn.Linear(emb_dim * 2, hidden)  # 128 → 128
```

**具体计算过程：**

```python
# 输入：拼接后的 128 维向量
combined = torch.cat([cell_emb, pert_vec], dim=-1)  # [64 + 64] = 128

# 第一层权重矩阵 W1: [128, 128], 偏置 b1: [128]
hidden_features = combined @ W1 + b1

# 每个输出维度的计算：
hidden[0] = combined[0]*W1[0,0] + combined[1]*W1[1,0] + ... + combined[127]*W1[127,0] + b1[0]
hidden[1] = combined[0]*W1[0,1] + combined[1]*W1[1,1] + ... + combined[127]*W1[127,1] + b1[1]
# ... (128个输出维度)
```

**关键洞察：** 每一行权重 `W1[i,:]` 就是一个**特征检测器**，学习识别特定的 cell-perturbation 交互模式。

#### 激活函数 GELU

```python
hidden_activated = GELU(hidden_features)
```

**GELU 的数学形式：**
```python
GELU(x) = x * Φ(x)  # Φ 是标准正态分布的累积分布函数
# 近似: GELU(x) ≈ 0.5 * x * (1 + tanh(√(2/π) * (x + 0.044715 * x³)))
```

**作用：** 提供平滑的非线性，让网络能学习复杂的交互关系。

#### 第二层线性变换 (128 → 200)

```python
nn.Linear(hidden, n_genes)  # 128 → 200
```

```python
# 权重矩阵 W2: [128, 200], 偏置 b2: [200]  
delta = hidden_activated @ W2 + b2

# 每个基因的扰动效应：
delta[gene_0] = hidden[0]*W2[0,0] + hidden[1]*W2[1,0] + ... + hidden[127]*W2[127,0] + b2[0]
delta[gene_1] = hidden[0]*W2[0,1] + hidden[1]*W2[1,1] + ... + hidden[127]*W2[127,1] + b2[1]
# ... (200个基因)
```

### 权重学习：如何捕获生物学模式

#### 训练过程中的优化

```python
# 假设一个具体的训练样本
cell_emb = [0.2, -0.1, 0.5, 0.8, ...]      # T细胞状态，64维
pert_vec = [0.8, 0.3, -0.2, 0.1, ...]      # drug_X扰动，64维  
true_delta = [-0.5, 2.1, 0.0, -1.2, ...]   # 真实的基因变化，200维

# 前向传播
combined = [0.2, -0.1, 0.5, 0.8, ..., 0.8, 0.3, -0.2, 0.1, ...]  # 128维
hidden = ReLU(combined @ W1 + b1)           # 128维
pred_delta = hidden @ W2 + b2               # 200维

# 损失计算
loss = MSE(pred_delta, true_delta)
# loss = mean((pred_delta - true_delta)²)

# 反向传播更新权重
W1 -= learning_rate * ∂loss/∂W1
W2 -= learning_rate * ∂loss/∂W2
```

#### 权重矩阵的生物学意义

**W1 矩阵的每一行学到了什么？**

假设训练后 W1 的某些行变成了：

```python
# W1 的第一行：可能学会了识别"免疫激活+药物抑制"的组合模式
W1[0,:] = [0.8, 0.2, -0.1, ..., -0.9, 0.7, 0.3, ...]
#          ↑ cell_emb部分 ↑     ↑ pert_vec部分 ↑

# 当 cell_emb 表示活跃的T细胞 AND pert_vec 表示免疫抑制剂时
# hidden[0] = large positive value → 检测到了特定的交互模式
```

**W2 矩阵的每一列对应一个基因：**

```python
# W2 的第一列：控制基因G0001的扰动量
W2[:,0] = [0.5, -0.2, 0.8, 0.0, ...]
#           ↑     ↑     ↑    ↑
#        hidden特征的权重，决定这个基因如何响应不同的交互模式
```

### 具体学习过程示例

#### 场景：T细胞 + 免疫抑制药物

**训练样本：**
```python
# 训练数据告诉模型：T细胞用药后，IL2基因下调，CTLA4上调
cell_type = "T_cell"      
cell_emb = [0.2, -0.1, 0.5, ...]    # T细胞特征
perturbation = "immunosuppressant"   
pert_vec = [0.8, 0.3, -0.2, ...]    # 免疫抑制剂特征

true_response = {
    "IL2": -2.0,     # 下调
    "CTLA4": +1.5,   # 上调  
    "CD28": -0.5,    # 轻微下调
    # ... 其他197个基因
}
```

**网络学习过程：**

第1步：**特征检测器形成**
```python
# W1 的某一行逐渐学会检测"T细胞+免疫抑制"模式
W1[42, :] 逐渐调整为 [0.0, 0.0, 0.8, ..., 0.9, 0.0, -0.7, ...]
#                      ↑T细胞标志位↑    ↑免疫抑制标志位↑

# 当这种组合出现时：
hidden[42] = high_activation  # 检测器激活
```

第2步：**基因调控学习**  
```python
# W2 学会将检测到的模式映射到基因调控
W2[42, IL2_index] = -0.8    # 第42个特征激活时，IL2下调
W2[42, CTLA4_index] = +0.6  # 第42个特征激活时，CTLA4上调
```

第3步：**组合效应**
```python
# 最终预测时：
delta[IL2_index] = hidden[42] * W2[42, IL2_index] + 其他特征贡献
                 = high_value * (-0.8) + ...
                 = 负值 → IL2基因下调 ✓
```

### 多模式交互的学习

#### 网络学会了复杂的条件逻辑

```python
# 不同的hidden神经元学会了不同的生物学"规则"

hidden[0]: 检测 "干细胞 + 分化信号" → 激活分化基因程序
hidden[1]: 检测 "癌细胞 + 化疗药物" → 激活凋亡通路  
hidden[2]: 检测 "神经元 + 神经毒素" → 抑制突触蛋白
hidden[3]: 检测 "免疫细胞 + 炎症因子" → 激活炎症反应
# ... 128个不同的生物学交互检测器
```

#### 基因层面的整合

每个基因的最终扰动量是所有相关交互的**加权组合**：

```python
# 例如：基因G0042(某个转录因子)的扰动量
delta[G0042] = hidden[0]*W2[0,G0042] +      # 干细胞分化的贡献
               hidden[1]*W2[1,G0042] +      # 凋亡通路的贡献  
               hidden[15]*W2[15,G0042] +    # 其他相关通路
               # ... 所有128个特征的贡献
               bias[G0042]                   # 基线偏置
```

### 为什么这种设计有效？

#### 分层抽象的威力

```
层次1: cell_emb + pert_vec → combined (直接拼接)
层次2: combined → hidden (学习交互模式)  
层次3: hidden → delta (映射到基因调控)
```

**生物学类比：**
- **层次1**：把"细胞状态"和"药物信息"放在一起
- **层次2**：细胞内的信号转导，识别出"这是什么情况"
- **层次3**：转录调控，决定"哪些基因该怎么变化"

#### 参数共享的学习效率

```python
# 同一套权重 W1, W2 处理所有的 (cell, perturbation) 组合
# 这意味着网络学会了通用的交互函数，而不是死记硬背每种组合
```

**优势：**
- 对于新的细胞-扰动组合，可以泛化预测
- 参数数量相对较少，训练效率高
- 学到的模式具有生物学可解释性

### 输出向量的生物学含义

#### 每个维度代表什么？

```python
delta = [∆gene0, ∆gene1, ∆gene2, ..., ∆gene199]  # 200维输出

# 例如：
delta[0] = -1.2   # 基因G0001的表达下调1.2个log单位  
delta[1] = +2.5   # 基因G0002的表达上调2.5个log单位
delta[2] = 0.0    # 基因G0003无明显变化
```

#### 与基础表达的结合

```python
# 最终预测的表达水平
predicted_expression = basal_expression + delta

# 具体计算：
final_G0001 = basal_G0001 + (-1.2)  # 在基础上下调
final_G0002 = basal_G0002 + (+2.5)  # 在基础上上调  
final_G0003 = basal_G0003 + (0.0)   # 保持不变
```

## 5. MLP基础原理详解

### MLP 的基本概念

#### 什么是感知器？

**感知器是最简单的神经网络单元**，模拟一个神经元：

```
输入 → [权重相乘] → [求和] → [激活函数] → 输出
```

**单个神经元的数学表达：**
```python
# 输入: x = [x₁, x₂, x₃, ..., xₙ]
# 权重: w = [w₁, w₂, w₃, ..., wₙ]  
# 偏置: b

output = activation(w₁×x₁ + w₂×x₂ + w₃×x₃ + ... + wₙ×xₙ + b)
       = activation(w·x + b)  # 向量点积形式
```

#### 多层感知器 = 多个单层感知器叠加

```
输入层 → 隐藏层1 → 隐藏层2 → ... → 输出层
```

### 具体的数学实现

#### 以项目中的 MLP 为例

```python
self.pert_dec = nn.Sequential(
    nn.Linear(emb_dim * 2, hidden),  # 第一层：128 → 128
    nn.GELU(),                       # 激活函数  
    nn.Linear(hidden, n_genes)       # 第二层：128 → 200

```

这是一个 **2层的MLP**，我们逐步拆解：

### 第一层：nn.Linear(128, 128)

#### 权重矩阵和偏置

```python
# PyTorch 内部创建的参数
W1 = torch.randn(128, 128)  # 权重矩阵 [输入维度, 输出维度]
b1 = torch.randn(128)       # 偏置向量 [输出维度]
```

#### 具体计算过程

**输入：** `x = [x₁, x₂, ..., x₁₂₈]` (128维向量)

**计算每个输出神经元：**
```python
# 第一个输出神经元
y₁ = x₁×W1[0,0] + x₂×W1[1,0] + x₃×W1[2,0] + ... + x₁₂₈×W1[127,0] + b1[0]

# 第二个输出神经元  
y₂ = x₁×W1[0,1] + x₂×W1[1,1] + x₃×W1[2,1] + ... + x₁₂₈×W1[127,1] + b1[1]

# ... 总共128个输出神经元
y₁₂₈ = x₁×W1[0,127] + x₂×W1[1,127] + ... + x₁₂₈×W1[127,127] + b1[127]
```

**矩阵形式（高效计算）：**
```python
# 向量化计算，一次性算出所有神经元
Y = X @ W1 + b1  # [1×128] @ [128×128] + [128] = [1×128]
```

#### 数值示例

```python
# 假设具体的输入和权重
X = [0.2, -0.1, 0.5, 0.8, ...]  # 128维输入

# 权重矩阵的第一列 (对应第一个输出神经元)
W1_col_0 = [0.3, -0.2, 0.7, 0.1, ...]  # 128个权重

# 第一个输出神经元的计算
y₁ = 0.2×0.3 + (-0.1)×(-0.2) + 0.5×0.7 + 0.8×0.1 + ... + 0.15
   = 0.06 + 0.02 + 0.35 + 0.08 + ... + 0.15
   = 1.23  # 假设最终结果
```

### 激活函数：GELU

#### 为什么需要激活函数？

**没有激活函数的话：**
```python
# 如果只是线性变换的堆叠
layer1 = X @ W1 + b1
layer2 = layer1 @ W2 + b2

# 等价于一个大的线性变换
layer2 = X @ (W1 @ W2) + (b1 @ W2 + b2) = X @ W_combined + b_combined
```

**结论：** 没有激活函数，多层神经网络就退化成了单层线性变换！

#### GELU 激活函数

```python
def gelu(x):
    # 高斯误差线性单元
    return x * 0.5 * (1.0 + torch.tanh(
        math.sqrt(2.0 / math.pi) * (x + 0.044715 * torch.pow(x, 3.0))
    ))
```

**特点：**
- 在 x>0 时近似线性，在 x<0 时平滑衰减
- 比 ReLU 更平滑，梯度更稳定

**作用示例：**
```python
# 第一层的输出
hidden = [1.23, -0.45, 2.1, -0.8, 0.0, ...]  # 128维

# 经过 GELU 激活
activated = [gelu(1.23), gelu(-0.45), gelu(2.1), gelu(-0.8), gelu(0.0), ...]
          = [1.18, -0.12, 2.09, -0.19, 0.0, ...]  # 128维
```

### 第二层：nn.Linear(128, 200)

#### 从特征到基因

```python
# 第二层参数
W2 = torch.randn(128, 200)  # [128输入特征, 200个基因]
b2 = torch.randn(200)       # 每个基因一个偏置
```

**计算过程：**
```python
# 输入是第一层激活后的128维特征
activated = [1.18, -0.12, 2.09, -0.19, 0.0, ...]

# 计算第一个基因(G0001)的扰动量
gene_0 = 1.18×W2[0,0] + (-0.12)×W2[1,0] + 2.09×W2[2,0] + ... + b2[0]

# 计算第二个基因(G0002)的扰动量  
gene_1 = 1.18×W2[0,1] + (-0.12)×W2[1,1] + 2.09×W2[2,1] + ... + b2[1]

# ... 总共200个基因
```

**矩阵形式：**
```python
delta = activated @ W2 + b2  # [1×128] @ [128×200] + [200] = [1×200]
```

### 完整的前向传播流程

#### PyTorch 代码实现

```python
class SimpleMLPDemo(nn.Module):
    def __init__(self):
        super().__init__()
        self.layer1 = nn.Linear(128, 128)
        self.activation = nn.GELU()  
        self.layer2 = nn.Linear(128, 200)
    
    def forward(self, x):
        # x: [batch_size, 128]
        
        # 第一层线性变换
        h1 = self.layer1(x)         # [batch_size, 128]
        
        # 激活函数
        h1_activated = self.activation(h1)  # [batch_size, 128]
        
        # 第二层线性变换  
        output = self.layer2(h1_activated)  # [batch_size, 200]
        
        return output
```

### 权重学习：反向传播

#### 梯度计算

```python
# 前向传播
pred_delta = mlp(combined_input)          # [1, 200]
loss = mse_loss(pred_delta, true_delta)   # 标量

# 反向传播  
loss.backward()

# 权重更新
with torch.no_grad():
    W1 -= learning_rate * W1.grad
    b1 -= learning_rate * b1.grad
    W2 -= learning_rate * W2.grad  
    b2 -= learning_rate * b2.grad
```

#### 权重的物理意义

**W1 矩阵：** 每一行是一个"特征检测器"
```python
# W1 的第42行可能学会了检测"T细胞+免疫抑制剂"的组合
W1[42, :] = [0.0, 0.0, 0.8, ..., 0.9, 0.0, -0.7, ...]
#           ↑ 对cell_emb的权重 ↑  ↑ 对pert_vec的权重 ↑
```

**W2 矩阵：** 每一列控制一个基因的调控
```python  
# W2 的第5列控制基因G0005的表达变化
W2[:, 5] = [0.3, -0.8, 0.1, ...]  # 128个特征对这个基因的贡献权重
```

### MLP 在项目中的具体作用

```python

def perturbation_effect(cell_state, perturbation_type):
```

```python
def forward_with_hooks(model, x):
    activations = {}

        def fn(module, input, output):
```

## 总结

### 关键设计思想

1. **模块化架构**：基础表达 + 扰动效应的加法分解
2. **交互建模**：通过MLP学习细胞-扰动的条件依赖关系  
3. **零样本能力**：利用embedding空间的结构化学习实现未见扰动的预测
4. **可解释性**：每个组件都有明确的生物学意义

### 技术优势

- **计算高效**：相比复杂生成模型，前向传播很快
- **参数共享**：同一套权重处理所有细胞-扰动组合
- **可扩展性**：模块化设计便于替换和升级
- **生物学友好**：符合生物学的层次化调控逻辑

### 局限性

- **线性假设**：加法组合可能过于简单
- **单一扰动**：无法处理多重扰动的交互效应
- **静态建模**：忽略时间动态和剂量效应

### 发展方向

- **架构升级**：替换为CPA/GEARS等更强模型
- **多模态融合**：整合蛋白组、表观组等数据
- **动态建模**：引入时间序列和剂量响应
- **机理整合**：与代谢、信号通路模型深度耦合

**MLP 就是一个函数逼近器**，通过多层的线性变换+非线性激活，学习从输入到输出的复杂映射关系。在这个虚拟细胞项目中，它成功地将128维的细胞-扰动组合信息转化为200维的基因调控指令，展现了深度学习在生物学建模中的威力。
