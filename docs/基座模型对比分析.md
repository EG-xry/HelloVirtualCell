## 1. 问题背景

在HelloVirtulCell项目的Stage 3扰动预测模块中，MLP需要接收细胞的向量表示作为输入。这个细胞表示来自于Stage 2的基座模型。

**关键问题：** 本项目中MLP的基座模型是什么？它与真实的单细胞基础模型有什么区别？

## 2. 项目内基座模型：CellFoundation

### 2.1 模型架构

```python
class CellFoundation(nn.Module):
    def __init__(self, n_genes=200, emb_dim=64, n_heads=4, n_layers=2):
        super().__init__()
        self.gene_emb = nn.Embedding(n_genes + 1, emb_dim)      # 基因嵌入
        self.expr_emb = nn.Embedding(n_bins + 1, emb_dim)       # 表达量嵌入  
        self.cls_token = nn.Parameter(torch.randn(1, 1, emb_dim) * 0.02)  # [CLS]标记
        
        # Transformer Encoder (2层，4头注意力)
        encoder_layer = nn.TransformerEncoderLayer(
            d_model=64, nhead=4, dim_feedforward=128,
            batch_first=True, activation="gelu"
        )
        self.encoder = nn.TransformerEncoder(encoder_layer, num_layers=2)
```

### 2.2 关键配置参数

```python
# 来自 config.py
EMB_DIM = 64                    # 嵌入维度
N_HEADS = 4                     # 注意力头数  
N_LAYERS = 2                    # Transformer层数
MAX_GENES_PER_CELL = 64         # 每个细胞的最大基因数量(序列长度)
EXPR_BINS = 8                   # 表达量离散化的bins数量
```

### 2.3 数据流水线

#### 细胞Token化
```python
def tokenize_cell(expr_row, max_genes=64):
    """将细胞转化为(gene_id, expr_bin)序列"""
    # 选择表达量最高的64个基因
    idx = np.argsort(-expr_row)[:max_genes]  
    vals = expr_row[idx]
    
    # 将连续表达量离散化为8个bins
    bins = np.floor(vals / vals.max() * EXPR_BINS)
    return gene_ids, expr_bins
```

**数据转换示例：**
```python
# 原始表达向量: [0.5, 2.1, 0.0, 1.8, ...] (200个基因)
# ↓ tokenize_cell
# gene_ids:  [1, 3, 0, 5, ...]     (top-64个高表达基因的ID)
# expr_bins: [2, 7, 1, 6, ...]     (对应的表达量离散化bin)
```

#### Transformer编码过程
```python
def forward(self, gene_ids, expr_bins):
    # 嵌入：基因ID + 表达量bin
    h = self.gene_emb(gene_ids) + self.expr_emb(expr_bins)  # (B, 64, 64)
    
    # 添加[CLS] token
    cls = self.cls_token.expand(b, -1, -1)                  # (B, 1, 64)  
    h = torch.cat([cls, h], dim=1)                          # (B, 65, 64)
    
    # Transformer编码
    h = self.encoder(h)                                     # (B, 65, 64)
    
    # 提取细胞嵌入
    cell_emb = h[:, 0]                                      # (B, 64) ← [CLS]位置
    return cell_emb
```

### 2.4 MLP如何使用基座模型

```python
# Stage 2: 生成细胞嵌入
cell_embs = embed_cells(foundation_model, adata.X)  # (2000, 64)

# Stage 3: MLP接收细胞嵌入
class CPAlite(nn.Module):
    def forward(self, cell_emb, pert_id):
        # cell_emb来自CellFoundation的输出！
        pert_vec = self.pert_emb(pert_id)                           # (64,)
        combined = torch.cat([cell_emb, pert_vec], dim=-1)          # (128,)
        delta = self.pert_dec(combined)                             # (200,)
        return delta
```

### 2.5 设计思想

- **类scGPT架构**: 使用(基因ID, 表达bin)序列表示细胞
- **Masked预训练**: 随机mask一些基因，预测被mask的表达量  
- **[CLS] token**: 类似BERT，用于提取全局细胞表示
- **生物学动机**: "一个细胞 = 一个句子，高表达基因 = 重要单词"

## 3. 真实单细胞基础模型介绍

### 3.1 scGPT

**官方资源：**
- **GitHub**: [bowang-lab/scGPT](https://github.com/bowang-lab/scgpt) 
- **论文**: [Nature Methods 2024](https://www.nature.com/articles/s41592-024-02201-0)
- **Virtual Cells平台**: [scGPT模型页面](https://virtualcellmodels.cziscience.com/model/scgpt)

**技术特点：**
- **训练数据**: 33+ million cells from CZ CELLxGENE
- **架构**: Generative Pretrained Transformer
- **发布状态**: 2024年Nature Methods正式发表
- **核心能力**: 
  - Multi-batch integration (批次效应校正)
  - Multi-omic integration (多组学整合)
  - Cell-type annotation (细胞类型注释)
  - Genetic perturbation prediction (遗传扰动预测)

**应用领域：**
```python
# scGPT的主要应用
- 批次校正：消除实验批次效应，保留生物学差异
- 多组学整合：整合scRNA-seq, scATAC-seq, 蛋白丰度数据
- 细胞注释：基于基因表达谱进行细胞类型注释
- 扰动预测：预测基因扰动对细胞状态的影响
- 基因网络推断：揭示基因-基因相互作用关系
```

### 3.2 Geneformer

**官方资源：**
- **Hugging Face**: [ctheodoris/Geneformer](https://huggingface.co/ctheodoris/Geneformer)
- **论文**: [Nature 2023](https://www.nature.com/articles/s41586-023-06139-9) 
- **Virtual Cells平台**: [Geneformer模型页面](https://virtualcellmodels.cziscience.com/model/geneformer)

**技术特点：**
- **训练数据**: 100+ million single-cell transcriptomes
- **预训练方法**: Masked gene prediction (15% masking rate)
- **数据表示**: Rank-value encoding by expression level
- **核心优势**: Self-supervised learning, context-aware predictions

**预训练策略：**
```python
# Geneformer的核心训练思想
# 1. 将每个细胞的转录组按表达量排序
# 2. 随机mask 15%的基因
# 3. 训练模型预测被mask位置的正确基因
# 4. 利用剩余基因的上下文信息进行推理

def pretraining_objective():
    """
    输入: [Gene_1, Gene_3, [MASK], Gene_7, Gene_2, ...]
    目标: 预测[MASK]位置应该是 Gene_5
    """
    masked_transcriptome = mask_genes(ranked_genes, mask_ratio=0.15)
    prediction = model(masked_transcriptome)
    loss = cross_entropy(prediction, true_genes)
    return loss
```

**应用场景：**
- **网络生物学**: Transfer learning用于有限数据的网络预测
- **基因功能预测**: 基于上下文推断基因功能
- **细胞状态转换**: 预测细胞发育轨迹和状态转换

### 3.3 UCE (Universal Cell Embeddings)

**官方资源：**
- **GitHub**: [snap-stanford/UCE](https://github.com/snap-stanford/UCE)
- **论文**: [bioRxiv 2023](https://www.biorxiv.org/content/10.1101/2023.11.28.568918v1)
- **Virtual Cells平台**: [UCE模型页面](https://virtualcellmodels.cziscience.com/model/uce)

**技术特点：**
- **模型规模**: 650+ million parameters, 33 layers
- **训练数据**: 36 million cells from 8 species
- **核心创新**: Zero-shot cross-species embedding
- **基因表示**: ESM2-tokenized genes (蛋白质语言模型)

**跨物种能力：**
```python
# UCE支持的物种
species = [
    "human",           # 人类
    "mouse",           # 小鼠  
    "zebrafish",       # 斑马鱼
    "mouse_lemur",     # 鼠狐猴
    "crab_eating_macaque",  # 食蟹猕猴
    "rhesus_macaque",  # 恒河猴
    "tropical_clawed_frog", # 热带爪蛙
    "pig"              # 猪
]

# 任何新细胞都可以映射到这个统一的嵌入空间
def zero_shot_embedding(new_cell_from_any_species):
    """无需额外训练或微调，直接生成细胞嵌入"""
    universal_embedding = UCE(new_cell_from_any_species)
    return universal_embedding  # 统一的生物学潜在空间
```

**性能优势：**
- 在zero-shot benchmarks上超越scGPT和Geneformer
- 总体得分优于Geneformer 13.9%
- 生物学保守性得分优于16.2%
- 批次校正得分优于10.1%

## 总结

### 当前CellFoundation的价值

**设计优势：**
- ✅ 轻量级，适合MVP演示
- ✅ 完全可控，便于学习和调试  
- ✅ 模块化设计，易于理解和修改
- ✅ 快速迭代，降低实验成本

**局限性：**
- ❌ 训练数据量小，泛化能力有限
- ❌ 模型规模小，表达能力不足
- ❌ 缺乏真实生物学先验知识
- ❌ 无法处理复杂的生物学现象

### 升级的必要性

随着项目从MVP向生产级系统演进，基座模型的升级是必然的：

1. **性能提升**: 真实基础模型在各项指标上都显著优于当前实现
2. **数据支持**: 支持真实单细胞数据，提高实用价值  
3. **生态对接**: 与主流生信工具链无缝集成
4. **研究价值**: 获得更有意义的生物学洞察

### 关键设计思想的传承

无论选择哪种升级路径，项目的核心设计思想都值得保持：
